# API Documentation

## Overview

The Customer Churn Prediction API provides REST endpoints for real-time customer churn prediction, health monitoring, and Prometheus metrics collection.

The API is implemented using FastAPI and serves the latest approved production model.

Base URL (Local)

```text
http://localhost:8000
```

---

# API Endpoints

| Method | Endpoint | Purpose                   |
| ------ | -------- | ------------------------- |
| GET    | /        | API Information           |
| GET    | /health  | Health Check              |
| POST   | /predict | Customer Churn Prediction |
| GET    | /metrics | Prometheus Metrics        |

---

# GET /

## Description

Returns a welcome message indicating that the API is running.

### Response

```json
{
  "message": "Customer Churn Prediction API"
}
```

---

# GET /health

## Description

Checks whether the API is operational.

This endpoint is used by Kubernetes readiness and liveness probes.

### Response

```json
{
  "status": "healthy"
}
```

### Status Codes

| Code | Meaning            |
| ---- | ------------------ |
| 200  | Service is healthy |

---

# POST /predict

## Description

Predicts whether a customer is likely to churn.

The endpoint accepts customer information as JSON and returns the prediction generated by the production model.

---

## Request Body

```json
{
  "Age": 35,
  "Gender": "Male",
  "Tenure": 24,
  "Usage Frequency": 18,
  "Support Calls": 2,
  "Payment Delay": 1,
  "Subscription Type": "Premium",
  "Contract Length": "Annual",
  "Total Spend": 1200,
  "Last Interaction": 8
}
```

---

## Successful Response

```json
{
  "prediction": 0
}
```

Where:

| Value | Meaning                           |
| ----- | --------------------------------- |
| 0     | Customer is not expected to churn |
| 1     | Customer is expected to churn     |

---

## Error Response

Example:

```json
{
  "detail": "Invalid input data."
}
```

Possible causes include:

* Missing required fields
* Invalid data types
* Feature mismatch
* Internal prediction errors

---

# GET /metrics

## Description

Exposes Prometheus metrics.

This endpoint is intended for Prometheus scraping rather than direct user interaction.

Example output:

```text
prediction_requests_total 152

prediction_errors_total 2

prediction_latency_seconds_bucket{le="0.1"} 145
```

---

# Request Processing Workflow

Every prediction request follows the sequence below.

```text
Client
   │
   ▼
POST /predict
   │
   ▼
Request Validation
   │
   ▼
Feature Engineering
   │
   ▼
Feature Scaling
   │
   ▼
Model Prediction
   │
   ▼
Return Response
   │
   ▼
Update Prometheus Metrics
```

---

# Production Artifacts Used

The API loads the following artifacts during startup:

| Artifact          | Purpose           |
| ----------------- | ----------------- |
| churn_model.pkl   | Production model  |
| scaler.pkl        | Feature scaling   |
| feature_names.pkl | Feature ordering  |
| baseline.json     | Drift monitoring  |
| metadata.json     | Model information |

These artifacts ensure that predictions remain consistent with the training pipeline.

---

# Error Handling

The API validates requests before inference.

Common validation includes:

* Missing fields
* Invalid feature values
* Incorrect data types
* Model loading failures
* Prediction exceptions

Meaningful HTTP responses are returned whenever an error occurs.

---

# Performance Considerations

The API is designed for low-latency inference.

Performance features include:

* Model loaded once during startup
* In-memory inference
* Lightweight request validation
* Automatic metrics collection
* Containerized deployment

---

# API Summary

The FastAPI service acts as the central interface between clients and the machine learning model.

Its primary responsibilities include:

* Serving production predictions
* Monitoring application health
* Exporting Prometheus metrics
* Supporting Kubernetes health checks
* Loading production artifacts
* Providing a reliable inference interface

The modular API design allows the application to scale easily while integrating seamlessly with Docker, Kubernetes, Prometheus, Grafana, and the automated retraining pipeline.
